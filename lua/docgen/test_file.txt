Docgen                                                                *docgen*

Full-featured Vimdoc generator for LuaCATs annotations. Simply run it with a
list of files to get properly tagged and formatted docs.

Supports the following features:
• Help tags are automatically generated based on the directory structure of
  the target files
• `@tag` annotations for defining additional helptags
• `@inlinedoc` and `@nodoc` to control display
• Automatic table of contents generation
• Descriptive text is parsed as markdown and automatically formatted,
  including line wrapping
• `@deprecated` tags allow for a one-line description
• Optional output logging
• Async file read and write

Requirements: ~

Neovim built with LuaJIT. Supported versions:
• Nightly
• Current (`0.12`) and previous release (`0.11`)

Installation: ~

Clone the repo.

CI Usage: ~

containing modified core code.

generate({paths}, {output}, {level}, {log_path})           *docgen.generate()*
    Main generator function

    Parameters: ~
        • {paths}    (`string[]`) Input filepaths
        • {output}   (`string?`) Output file
        • {level}    (`integer?`) Log level
            • 0 Standard output only
            • 1 Standard messages
            • 2 Debug messages
        • {log_path} (`string?`) Log path

Docgen.test_file                                            *docgen.test_file*

This is a brief. It allows the user to explain additional things in the
documentation that are not necessarily relevant to any particular function or
class. It's a good way to provide an overview and introduction to the module.
